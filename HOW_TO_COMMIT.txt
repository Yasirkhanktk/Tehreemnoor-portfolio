# 🚀 HOW TO COMMIT TO GIT (Copy & Paste Commands)

## Method 1: Create New Repository "figma-ai-changes"

### Step 1: Initialize Git (if not already done)
```bash
git init
```

### Step 2: Add All Files
```bash
git add .
```

### Step 3: Commit with Message
```bash
git commit -m "feat: integrate Supabase database for portfolio

✅ Added Supabase integration
✅ Migrated from localStorage to PostgreSQL  
✅ Full CRUD operations via Supabase API
✅ Added async error handling
✅ Created SQL schema for database
✅ Added 10 sample projects
✅ Environment variables setup
✅ Complete documentation

Files changed:
- NEW: /src/lib/supabaseClient.ts
- UPDATED: /src/app/admin/dataStore.ts
- UPDATED: ManageProjects/Testimonials/Experience
- NEW: /supabase-schema.sql
- NEW: Documentation files"
```

### Step 4: Create GitHub Repository
1. Go to github.com
2. Click "New repository"
3. Name: `figma-ai-changes`
4. Don't initialize with README
5. Click "Create repository"

### Step 5: Connect and Push
```bash
git remote add origin https://github.com/YOUR_USERNAME/figma-ai-changes.git
git branch -M main
git push -u origin main
```

---

## Method 2: Push to Existing Repository

### If you already have a git repo:
```bash
git add .
git commit -m "feat: integrate Supabase database"
git push
```

---

## Method 3: Using Git GUI

### If you prefer a visual interface:
1. Open GitHub Desktop or GitKraken
2. Select "Add all files"
3. Write commit message:
   ```
   feat: integrate Supabase database for portfolio
   
   - Added Supabase client
   - Updated all CRUD operations
   - Added SQL schema
   - Complete documentation
   ```
4. Click "Commit to main"
5. Click "Push origin"

---

## ✅ Verify Your Commit

After pushing, check:
- [ ] All files uploaded to GitHub
- [ ] Commit message visible on GitHub
- [ ] `.env.local` is NOT in the repo (should be ignored)
- [ ] Documentation files are there

---

## 📝 What Gets Committed:

✅ Source code files
✅ Documentation (*.md files)
✅ SQL schema file
✅ .env.local.example (template)
✅ package.json (with Supabase dependency)

❌ .env.local (your actual API keys - stays local)
❌ node_modules/ (dependencies)
❌ .DS_Store, etc.
